### 🐛 Issues & Learnings

#### Issue: Markdown Rendering Failure
- **Root Cause**: Typography plugin was never installed during initial project setup. The `prose` utility classes were being used throughout the codebase but had no CSS definitions, causing blog posts to display as unstyled plain text. The MarkdownEditor component's inline preview styles created false confidence that posts would render correctly in production.

- **Fix**:
  - Installed `@tailwindcss/typography@^0.5.19` package
  - Configured plugin in `tailwind.config.ts`
  - Verified build and user-tested in browser
  - Created automated verification to prevent recurrence

- **Prevention**:
  - Implemented dependency verification script that runs on demand
  - Documented all critical dependencies in CLAUDE.md
  - Added pre-deployment checklist requiring visual markdown testing
  - Created exhaustive post-mortem for future reference

- **Time Impact**: ~2 hours total (root cause analysis: 30min, fix implementation: 20min, documentation: 70min)
