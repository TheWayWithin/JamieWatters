### Issues & Learnings

#### Issue: Prisma Migration Conflict
- **Root Cause**: Database had modified migration history that conflicted with new migration
- **Fix**: Used `npx prisma db push` instead of `migrate dev` to sync schema directly
- **Prevention**: For development databases, `db push` is faster; use migrations only for production

#### Issue: ACTIVE Enum Value Missing
- **Root Cause**: Database contained legacy ACTIVE status not defined in schema
- **Fix**: Added `ACTIVE // Legacy value` to ProjectStatus enum for backward compatibility
- **Prevention**: Always check existing data before modifying enums

#### Issue: TypeScript Errors with New Fields
- **Root Cause**: ProjectForm and admin page interfaces didn't include projectType field
- **Fix**: Added projectType and customMetrics to all relevant interface definitions
- **Prevention**: When adding database fields, grep for all usages of the model interface
